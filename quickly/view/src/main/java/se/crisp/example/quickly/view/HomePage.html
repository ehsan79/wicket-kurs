Here you start your journey

This is a starting plate for your Wicket with Spring repository project. Also included is Twitter Bootstrap, the CSS framework and Lombok which gives us auto-generated getters, setters, equals, hashcode and toString.

There are two maven modules, Core and View, which have a common parent: Quickly.

Core is the backend which is configured by the CoreConfiguration class. There you will find that if there is no database in the JNDI, a fall back to an in-memory database will happen.

The View module is for your practicing of Wicket. Here is a BasePage and a HomePage, the latter inherits from the former. You are currently reading HomePage. The HomePage logic is found in the class with the same name.

Start this as a web application

To start this you could either build and install to your own Tomcat with a proper MySQL database or run everything in a Jetty container with an in-memory database.

Running in IntelliJ

With IntelliJ, browse the Maven structure, go to the view module, go to Plugins, go to jetty and right-click on jetty:run. Select "Debug view [[jetty:run]]". The web application should start at localhost:8080. Selecting debug instead of run gives you hot deploy on build (Ctrl-F9) and fewer restarts of the container.
